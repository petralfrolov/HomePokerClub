# Запуск Home Poker Club через Docker

В проекте настроена удобная Docker-контейнеризация, которая позволяет запускать приложение как в **продакшене (на сервере)**, так и **локально для отладки и разработки**, без необходимости вносить какие-либо изменения в код приложения.

---

## 1. Архитектура Docker-окружения

При запуске через Docker приложение состоит из двух сервисов, объединенных в виртуальную сеть Docker:
1. **`backend` (FastAPI)**:
   - Запускается на порту `8000` внутри сети Docker.
   - База данных SQLite сохраняется в постоянном томе (volume) `poker_db_data`.
   - При старте контейнера автоматически запускаются новые миграции БД через `alembic upgrade head`.
2. **`frontend` (Nginx + React SPA)**:
   - Собирает оптимизированный production-билд фронтенда.
   - Запускает легковесный веб-сервер Nginx на порту `80`.
   - Автоматически проксирует запросы на API (`/api`, `/docs`, `/openapi.json`, `/static`) и WebSocket-соединения (`/ws`) к контейнеру бэкенда.

---

## 2. Локальная сборка и публикация образов в Docker Hub

Чтобы развернуть приложение на сервере без переноса исходного кода, вы можете собрать образы локально на рабочем компьютере, опубликовать их в своем профиле Docker Hub, а на сервере просто стянуть готовые образы.

По умолчанию имя пользователя Docker Hub установлено в `petralfrolov`. Если ваш логин отличается, вы можете переопределить его в `.env` файле, задав переменную `DOCKER_REGISTRY_USER=ваш_логин`.

### Шаг 1. Авторизация в Docker Hub
Убедитесь, что Docker Desktop запущен, и выполните в терминале локально:
```bash
docker login
```

### Шаг 2. Сборка образов
Соберите образы с тегами вашего репозитория:
```bash
docker compose build
```

### Шаг 3. Публикация в Docker Hub
Отправьте собранные образы в публичный доступ:
```bash
docker compose push
```

## 3. Деплой и запуск на сервере (Production режим)

Главный плюс такого подхода — **на сервере не нужны файлы исходного кода, Node.js или компиляторы**. Вам понадобятся всего два файла: `docker-compose.prod.yml` и `.env`.

### Шаг 1. Подготовка файлов на сервере
Создайте на сервере папку (например, `~/poker`) и скопируйте туда файлы:
1. [docker-compose.prod.yml](file:///d:/Python/PythonProjects/HomePokerClub/docker-compose.prod.yml) (переименуйте его на сервере в `docker-compose.yml`)
2. Создайте файл `.env` на сервере:
   ```bash
   # ~/poker/.env
   # Укажите ваш логин Docker Hub, чтобы Docker знал откуда скачивать образы
   DOCKER_REGISTRY_USER=petralfrolov
   
   # Разрешенные адреса для CORS (если требуется)
   CORS_ORIGINS=http://your-server-ip,http://localhost
   ```

### Шаг 2. Запуск проекта на сервере
Выполните команду в папке с файлами на сервере:
```bash
docker compose up -d
```
Docker автоматически:
1. Скачает готовые образы бэкенда и фронтенда из вашего репозитория Docker Hub.
2. Создаст постоянный том (Volume) для базы данных.
3. Проверит и применит новые миграции базы данных бэкенда.
4. Запустит фронтенд на порту `80` сервера.

### Шаг 3. Управление проектом на сервере

- **Посмотреть статус работы:**
  ```bash
  docker compose ps
  ```
- **Просмотр логов в реальном времени:**
  ```bash
  docker compose logs -f
  ```
- **Остановка приложения:**
  ```bash
  docker compose down
  ```
- **Обновление приложения до новой версии:**
  Когда вы опубликуете новые версии образов локально через `docker compose push`, на сервере достаточно выполнить:
  ```bash
  docker compose pull
  ```
  и затем перезапустить контейнеры:
  ```bash
  docker compose up -d
  ```

## 4. Локальный запуск для отладки новых изменений (Development режим)

При отладке новых изменений на локальной машине вам нужна **горячая перезагрузка (Hot Reload)**, чтобы изменения в коде мгновенно отображались в браузере.

Для этого используйте классический локальный запуск без контейнеров:

### Запуск в одно нажатие (Windows PowerShell):
Просто запустите скрипт из корня проекта:
```powershell
./run_app.ps1
```
Этот скрипт:
1. Запустит локальный бэкенд FastAPI на порту **`8001`** с флагом `--reload` (любое изменение Python-кода перезапустит сервер).
2. Запустит фронтенд Vite в режиме разработки на порту **`5173`**.
3. Настроит проксирование Vite так, что все `/api` и `/ws` запросы будут автоматически лететь на локальный бэкенд.
